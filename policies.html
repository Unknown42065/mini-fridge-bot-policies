The Fridge Bot - Privacy Policy

**Last Updated:** September 27, 2025

This Privacy Policy describes how **The Fridge** bot ("The Bot") collects, uses, and stores data from Discord users and servers.

1. Data Collection and Use

The Bot only collects and stores data that is strictly necessary for its security, moderation, and core functionality.

1.1 Server Data (Required for Operation)

Data Collected Purpose Retention Period
**Server IDs (Guild IDs)** To identify the server, locate the log channel, and apply server-specific settings (e.g., quarantine state). Retained while The Bot is in the server.
**Channel IDs & Role IDs** To correctly configure and apply moderation actions (e.g., log channel, quarantined role, member count channel). Retained while The Bot is in the server.

1.2 User Data (Required for Moderation and Security)

Data Collected Purpose Retention Period
**User IDs** To track user activity for **spam detection** (spam_detector map), track strikes for **NSFW offenses** (NSFW_STRIKES map), and maintain the **Whitelist** (WHITELISTED_USERS list). **Temporary:** Spam and activity data is cleared within 10-15 seconds. **Persistent:** NSFW strike counts and Whitelist IDs are stored indefinitely until manually cleared by an administrator or the user leaves the server.
**Message Content/URLs** To check messages for **spam** (rate limiting), detect **ghost pings**, and send **image/GIF URLs** to the **Sightengine API** for NSFW filtering. **Ephemeral:** Message content is not logged or stored, except for the **Sightengine API** check, which uses the URL only. Message data for ghost ping detection is deleted from the Bot’s internal cache within seconds.

2. Third-Party Data Sharing

The Fridge utilizes the **Sightengine API** for advanced NSFW and content filtering.

  • When a user sends a message with an image or GIF URL, that **URL** (not the user ID or server ID) is sent to the Sightengine API for analysis.
  • The results of the analysis (e.g., a high score for "sexual activity") are used only to determine if the message should be deleted and if a strike should be applied.
  • We do not share any other user data (IDs, names, messages) with Sightengine or any other third party.

3. Data Deletion and Removal

3.1 Server Removal

If The Bot is removed from a Discord server, all associated Server IDs, Channel IDs, Role IDs, and the server's specific configuration data will be **automatically deleted** from our storage.

3.2 User Data Removal

  • **Administrators** can manually remove a user's persistent data (NSFW strikes) using the /clear-strikes command.
  • User IDs stored in the Whitelist can be removed using the /remove-whitelist command.
  • Ephemeral data (spam history, ghost ping cache) is **automatically deleted** in real-time.

4. Contact Information

If you have any questions, concerns, or requests regarding this Privacy Policy or the data collected by The Fridge, please contact the developer via Discord or through the support channels provided on the server where the Bot is hosted.
